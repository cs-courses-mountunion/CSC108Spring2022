Assignment:
Modify Example 7-9 from the course textbook so that two images move randomly around the output window.  The program should respond to mouse clicks and key presses by repositioning the images to certain screen locations.
Sample:
Your program should work in a manner similar to this sample.  Right-click on the link in the preceding sentence and save the file in your CSC108 folder.  Then using File Explorer, right-click on the .zip file and choose "Extract All" from the pop-up menu.  After extracting the files from the .zip file, there will be a file with a .exe extension – double-click on the file to run the demo and see how your program should work.

Run the demo program, then press any key on the keyboard, then click the left mouse button, then click the right mouse button.  Notice how the images are repositioned when a key is typed or a mouse button is clicked.
Starting Point:
To get started on this assignment, make a new Processing sketch, then paste the following code into the editor window.  This code is a modified version of the code from Example 7-9 from the textbook. 
float speed = 5.5;
float x;
float y;

void setup() {
  size(800, 600); 
  smooth();
  x = width-100;
  y = 0;
}

void draw() {
  background(0);  
  x += random(-speed, speed);
  y += random(-speed, speed);
  x = constrain(x, 0, width);
  y = constrain(y, 0, height);
  fill(255, 0, 0);
  rect(x, y, 100, 100);
}
When you run this program, you will see a red square moving randomly around the screen, starting from the upper-right corner of the output window.
Your Task, Step 1:  Add a second square
First of all, you should change the program so that there are two squares moving on the screen, a red one and a blue one, with the blue one starting at the lower-left corner of the output window.  To do this, you will need to declare two more variables at the top of the program to store the x and y positions of the second square (maybe you could call these variables x2 and y2, or you can use other names you would like).  These two variables should be set to their proper starting values in your setup function.  You can copy all of the code in the draw function except the "background(0);" statement and then paste a copy of that code at the end of draw — modify the copied code so that it uses the two new variables you added, and so that the second square drawn is blue instead of red.
Your Task, Step 2:  Replace the squares with images
Find two images you like and draw them instead of the rectangles in your draw function.  Visit http://www.openclipart.org and search for two images that you like.  When you find an image that you would like to use, click on it, and then click on the "PNG" button on the web page.  This will cause a large version of the image to appear in your browser window — right-click the mouse on the image and then choose "Save Image As..." from the pop-up menu.  Save the image in the folder for your Processing program, and you probably want to give the file a simple name, since many of the names of the files on openclipart.org are long and somewhat weird.  For the example program, the names "penguin.png" and "piano.png" were used.

When you have the two images saved, use the "Add Files" option of the "Sketch" menu to add them to your program, and then replace each of the fill and the rect statements in your draw function with an image statement that draws each of your images in a 100 × 100 area.
Your Task, Step 3:  Add the ability to respond to user input
As the last steps in your draw function, add the following processing logic:

if the user presses any key on the keyboard,
set the x- and y-position of both images to the middle of the screen
if the user clicks the LEFT mouse button,
set the x- and y-positions of the first image so that the image is displayed at the top middle of the window, and set the x- and y-positions of the second image so that the image is displayed at the bottom middle of the window
if the user clicks the RIGHT mouse button,
set the x- and y-positions of the first image so that the image is displayed at the left middle of the window, and set the x- and y-positions of the second image so that the image is displayed at the right middle of the window
